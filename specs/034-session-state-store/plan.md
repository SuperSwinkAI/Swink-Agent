# Implementation Plan: Session Key-Value State Store

**Branch**: `034-session-state-store` | **Date**: 2026-03-31 | **Spec**: [spec.md](spec.md)
**Input**: Feature specification from `/specs/034-session-state-store/spec.md`

## Summary

Add a session-attached key-value state store (`SessionState`) with typed get/set, delta tracking, thread-safe concurrent access via `std::sync::RwLock`, and persistence through the existing `SessionStore`/`JsonlSessionStore` infrastructure. The state is accessible from tools, policies, and the agent owner. Deltas are flushed per-turn and emitted as `StateChanged` events. Full-snapshot persistence strategy (not delta replay).

## Technical Context

**Language/Version**: Rust latest stable (edition 2024)
**Primary Dependencies**: serde, serde_json, tokio, std::sync::RwLock (no new external deps)
**Storage**: JSONL via swink-agent-memory crate (extends existing SessionStore)
**Testing**: cargo test --workspace
**Target Platform**: Library crate (cross-platform)
**Project Type**: Library
**Performance Goals**: State operations are in-memory HashMap lookups вЂ” sub-microsecond. No hot-path concerns.
**Constraints**: Zero overhead for agents that don't use state. Backward compatible with pre-034 sessions/checkpoints.
**Scale/Scope**: State expected to be small (dozens of keys, not millions). No size limits enforced.

## Implementation Phases

### Phase 1: Core Types (FR-001 through FR-011, FR-023, FR-024)

Create `src/state.rs` with `SessionState` and `StateDelta`. Pure data structures with no external dependencies beyond `serde_json`. All unit tests for get/set/remove/contains/keys/len/clear/delta/flush_delta/collapse semantics.

**Files**: `src/state.rs` (new), `src/lib.rs` (re-export)

### Phase 2: Agent Integration (FR-012, FR-013, FR-022)

Add `Arc<RwLock<SessionState>>` to `Agent` struct. Add `session_state()` accessor. Add builder methods `with_initial_state()` and `with_state_entry()` to `AgentOptions`. Thread the state through to `AgentLoopConfig`.

**Files**: `src/agent.rs`, `src/agent_options.rs`

### Phase 3: Tool Access (FR-014)

Add `state: Arc<RwLock<SessionState>>` parameter to `AgentTool::execute`. Update all built-in tool implementations (`BashTool`, `ReadFileTool`, `WriteFileTool`). Update `MockTool` in test helpers. Update tool dispatch in the loop to pass state.

**Files**: `src/tool.rs`, `src/builtin_tools/*.rs`, `tests/common/mod.rs`, `src/loop_/mod.rs`

### Phase 4: Policy Integration (FR-015)

Add `state: &SessionState` to `PolicyContext`. Update all policy slot runners to acquire read lock and pass state reference. Update all built-in policy implementations in `swink-agent-policies` crate.

**Files**: `src/policy.rs`, `src/loop_/mod.rs`, `policies/src/*.rs`

### Phase 5: Loop & Events (FR-016, FR-017)

Add delta flush after PostTurn policy evaluation. Add `StateChanged` variant to `AgentEvent`. Add `state_delta: Option<StateDelta>` to `TurnSnapshot`. Emit `StateChanged` event when delta is non-empty.

**Files**: `src/loop_/mod.rs`, `src/types.rs`

### Phase 6: Checkpoint Support (FR-021)

Add `state: Option<Value>` with `#[serde(default)]` to `Checkpoint` and `LoopCheckpoint`. Update `save_checkpoint`/`restore_from_checkpoint` to include state. Update `pause`/`resume_stream` to include state.

**Files**: `src/checkpoint.rs`, `src/agent.rs`

### Phase 7: Session Persistence (FR-018, FR-019, FR-020)

Add `save_state`/`load_state` default methods to `SessionStore` trait. Implement state line format in `JsonlSessionStore`. Handle backward compatibility (missing state line = empty state).

**Files**: `memory/src/store.rs`, `memory/src/jsonl.rs`

### Phase 8: Integration Tests & Cleanup

End-to-end tests: tool writes state в†’ policy reads state в†’ delta emitted в†’ session saved в†’ session loaded в†’ state restored. Concurrent tool access stress test. Pre-034 backward compatibility test. Update any downstream code in TUI, eval, adapters, integration tests that implements `AgentTool`.

**Files**: `tests/state_tests.rs` (new), various downstream tool implementations
