# Implementation Plan: Gemma 4 Local Default (Direct Inference)

**Branch**: `041-gemma4-local-default` | **Date**: 2026-04-04 | **Spec**: [spec.md](spec.md)
**Input**: Feature specification from `/specs/041-gemma4-local-default/spec.md`

## Summary

Add direct in-process Gemma 4 inference to the `swink-agent-local-llm` crate, making Gemma 4 E2B the default local model (replacing SmolLM3-3B). This requires upgrading mistral.rs from 0.7 to 0.8, adding a `MultimodalModelBuilder` code path for Gemma 4 architecture, implementing a stateful cross-chunk thinking parser for Gemma 4's `<|channel>thought` delimiter format, and injecting the `<|think|>` control token for direct inference thinking mode. All changes are feature-gated behind `gemma4` until upstream stability is confirmed. The NaN logits bug (mistral.rs #2051) was reported against MoE variants (26B) with BF16/UQFF quantization — E2B (dense architecture) with Q4_K_M GGUF is likely unaffected. Implementation proceeds for E2B with a live validation gate after US1. E4B/26B (MoE) presets are deferred until the upstream fix ships.

## Design Decisions

### D1: Builder Branching in ChatBackend::build()

Branch on `ModelConfig::is_gemma4()` within the existing `ChatBackend::build()` method. Both `GgufModelBuilder` and `MultimodalModelBuilder` return `mistralrs::Model`, so the `LoaderBackend` trait, `LazyLoader`, and all downstream code remain unchanged.

```
ChatBackend::build(config)
  ├── config.is_gemma4() == true  → MultimodalModelBuilder::new(repo_id, vec![filename]).build()
  └── config.is_gemma4() == false → GgufModelBuilder::new(repo_id, vec![filename]).build()
```

### D2: Parser Dispatch

The `StreamState::process_content_delta()` method currently calls `extract_thinking_delta()` unconditionally. For Gemma 4, it dispatches to `ChannelThoughtParser` instead. The parser selection is determined once at `StreamState` construction time based on model family (not per-chunk).

The `ChannelThoughtParser` is a field on `StreamState` (behind `#[cfg(feature = "gemma4")]`). When the feature is disabled, the code path compiles away entirely.

### D3: Thinking Token Injection

`<|think|>` is prepended to the system prompt in `LocalConverter::system_message()` when:
1. The model is Gemma 4 (`ModelConfig::is_gemma4()`)
2. Thinking is enabled (determined via `ModelSpec` capabilities or explicit config)

This requires threading model config information into the converter. The current `convert_context_messages(context)` signature gains a `config: &ModelConfig` parameter.

### D4: Feature Gate Conditional Default

When `gemma4` feature is enabled: `DEFAULT_LOCAL_PRESET_ID = "gemma4_e2b"`.  
When `gemma4` feature is disabled: `DEFAULT_LOCAL_PRESET_ID = "smollm3_3b"` (unchanged).

This is implemented via `#[cfg(feature = "gemma4")]` on a constant definition with a fallback.

### D5: Tool Call Parser (P3, deferred)

The `ToolCallParser` will follow the same stateful pattern as `ChannelThoughtParser` — a struct on `StreamState` that processes each chunk and emits tool call events. It parses `<|tool_call>call:{name}{args}<tool_call|>` format. Implementation deferred until core text + thinking is stable.
